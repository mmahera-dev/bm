About

I started out writing Java on a hand-me-down laptop, learning the hard way that inheritance hierarchies get ugly fast if you don't respect the interfaces underneath them. That lesson stuck — most of my work now is about drawing clear boundaries between systems so they can change independently.

Over the last five years I've worked across backend services, cloud infrastructure, and the occasional compliance-heavy project where the code had to satisfy both an engineering review and a legal one. That mix taught me to write systems that are not just correct, but explainable.

Outside of shipping features, I spend time reading up on regulatory frameworks that touch software — data protection, IT law, that sort of thing — because the constraints outside the codebase shape the ones inside it.
